## Диагностика

Rolling window сервис работает в памяти, но нет способа получить доступ к данным скользящего окна. Пользователь хочет иметь возможность получать данные bid/ask за последний час.

## Предлагаемое изменение

Добавить REST API endpoint для получения данных rolling window:

1. Добавить метод в RollingWindowService для получения данных по exchange/symbol
2. Создать контроллер RollingWindowController с endpoint'ом GET /api/rolling-window/{exchange}/{symbol}
3. Зарегистрировать контроллер в DI

## Обоснование

Rolling window данные нужны для анализа и отображения. API endpoint позволит клиентам получать актуальные данные за последний час.

## Оценка рисков

- Низкий риск: добавление нового endpoint'а
- Возможна нагрузка на память при большом количестве запросов

## План тестирования

1. Запустить приложение
2. Сделать запрос к новому endpoint'у
3. Проверить, что возвращаются данные за последний час

## План отката

Удалить контроллер и endpoint из DI.
